# Question Guidelines: Symbolic & Structural (SSB2)

## What makes a good SSB2 question?
*   **Strict Syntax:** JSON, XML, Python code must be parsable.
*   **Transformation:** Convert CSV to JSON.
*   **Logic in Code:** "What does this function return?"

## What to avoid
*   **Pseudo-code:** Use executable languages.
*   **Ambiguous Specs:** The schema must be exact.

## Bias & Neutrality
*   **Language Agnosticism:** Test Python, JS, Rust, SQL.

## Difficulty Scaling
*   **Level 1:** Fix a missing semicolon.
*   **Level 2:** Write a function from spec.
*   **Level 3:** Refactor complex legacy code.
